Project context: Submitted as a BCA mini-project at Graphic Era Hill University, Dehradun (May 2026). Supervised by Mrs. Nidhi Joshi, School of Computing. The system is built for academic, lab, and small-organization environments where commercial SIEM tools are not practical.
🔍
Auto Format Detection
Automatically identifies system logs vs Apache / Nginx access logs using pattern matching before applying the correct parser.
🛡️
Rule-based Threat Engine
Detects brute-force login attempts, port scans, directory traversal, SQL injection indicators, and critical system failures.
📊
Visual Dashboard
Flask-powered browser interface with stat cards, risk score, charts, suspicious IP tables, and downloadable text reports.
⚡
Pandas-powered Analysis
Vectorized DataFrame operations enable processing of half a million log lines in under 8 seconds on standard hardware.
🧩
Modular Architecture
Each stage — parsing, analysis, charting, reporting — is an independent module. Easy to extend, test, and explain in a viva.
🎯
Heuristic Risk Score
A 0–100 composite score derived from brute-force sources, intrusion attempts, critical events, and error volume. Instant severity read.
Architecture

Analysis Pipeline

Every log file flows through a deterministic six-stage pipeline before results appear in the dashboard.

01
File Input
.log / .txt uploaded via Flask or CLI
02
Format Detection
Regex probe identifies sys-log or access-log
03
Regex Parsing
Named capture groups → structured records
04
DataFrame
Pandas stores + enriches with derived flags
05
Threat Detection
Rule engine scans brute-force, scans, injection
06
Output
Charts + report + dashboard render
Derived DataFrame Columns
timestamp level ip_address method path status_code message is_error is_warning is_failed_login is_critical hour date
Supported Log Formats
✓ System logs — timestamp / level / IP / message
✓ Apache access logs (Combined Log Format)
✓ Nginx access logs
✗ Windows Event Logs (future)
✗ JSON / cloud logs (future)

Security Engine

    
Threat Detection

    
Four rule-based detectors run over the enriched DataFrame. Each is transparent and auditable — no black-box ML required.


    

      

        
🔐

        

          
Brute-force Detection

          
Groups failed login records by IP address, counts attempts, and flags any source that exceeds the configurable threshold. Zero false positives on test data.

        

      

      

        
📡

        

          
Port Scan Detection

          
Searches log messages for known scan-related pattern strings. Alerts when reconnaissance indicators appear in rapid succession from a single source.

        

      

      

        
💉

        

          
Intrusion Detection

          
Matches suspicious request strings: directory traversal (../), CGI exploit paths, and SQL injection indicators in URL parameters and query strings.

        

      

      

        
🚨

        

          
Critical Event Detection

          
Identifies CRITICAL level entries and severe failure keywords such as kernel panics, disk errors, and OOM kills. Highlights high-priority items at the top of the dashboard.

        

      

    


    

      Risk Score: A heuristic 0–100 score is computed from the weighted count of brute-force sources, port-scan events, intrusion attempts, critical events, and general error volume. It provides an instant severity summary without requiring a formal threat model.

Core Module Responsibilities

    

      

        
LogParser

        
Reads files, detects format, parses lines, creates DataFrame with all raw and enriched columns.

      

      

        
LogAnalyzer

        
Performs statistical analysis and runs all four threat detection rule engines against the DataFrame.

      

      

        
ChartGenerator

        
Generates bar, line, and pie charts using Matplotlib. Outputs PNG files consumed by the dashboard.

      

      

        
ReportGenerator

        
Compiles a plain-text summary report with threat tables, statistics, and timestamp. Downloadable from dashboard.

      

      

        
app.py

        
Flask entry point — handles file uploads, runs the analysis pipeline, and renders the results dashboard via Jinja2 templates.
